## 17:03-17:19 CST — 上下文爆炸 & Hermes 记忆系统 研究

### 上下文爆炸问题讨论
- 原因：Prompt 过于冗长、多轮对话 context 累积、没有设定清晰边界
- 解决方案：精简 Prompt、分步处理、主动重启会话、外置存储、RAG
- 多 Agent 不能根本解决 context 爆炸，但可以解耦任务

### Hermes 记忆系统调研（重要！）

**核心发现**：Hermes 有三层记忆架构 + 两个关键操作

| 层次 | 类型 | 内容 | 存储 |
|------|------|------|------|
| Episodic | 情景记忆 | 对话历史 | sessions/ |
| Semantic | 语义记忆 | 用户偏好、决定、事实 | MEMORY.md / USER.md |
| Procedural | 程序记忆 | 学会的技能 | SKILL.md |

**两个关键操作**：
- **Recall（召回）**：用户提问前 → 搜索相关记忆 → 注入 context
- **Sync（同步）**：对话结束后 → 提取重要信息 → 写入记忆文件

**Hermes 的主动触发**：在 system prompt 里明确要求 AI 在用户提到过去事项时主动搜索历史
