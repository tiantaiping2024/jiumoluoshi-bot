## 进行中任务

### Codex + CC Switch + MiniMax 问题（核心阻塞）
- **症状**: `CC Switch local proxy failed while handling Codex endpoint /responses. Provider: MiniMax; model: ark-code-latest; upstream_status: HTTP 400; cause: invalid params, context window exceeds limit (2013)`
- **注意**: 错误是 context window exceeds，不是 unknown model
- **CC Switch catalog** 已添加 `ark-code-latest` 条目到 `~/.codex/cc-switch-model-catalog.json`
- **问题根因**: CC Switch 的 `model_mapper.rs` 把 `ark-code-latest` 直接作为模型名发给 MiniMax API，但 MiniMax API 只认 `MiniMax-M3`
- **关键**: CC Switch 的模型映射逻辑是硬编码在二进制中的，没有找到外部配置文件
- **已知可行方案**: Codex++（使用 litellm 做模型名翻译，可以把 `ark-code-latest` 映射为 `MiniMax-M3`）

### 待确认

- 二进制中 `model_mapper.rs` 是否读取外部 JSON 配置？
