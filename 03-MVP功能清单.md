# 鸠摩罗什Bot - MVP功能清单

## MVP (Minimum Viable Product) 概述

**MVP目标**: 验证核心价值主张，获取早期用户，建立产品口碑  
**MVP发布时间**: T+1个月  
**MVP核心假设**: 用户愿意与AI佛学助手进行持续对话，获取佛学知识和禅修指导

---

## P0 - 核心功能 (必须完成)

### 1. 基础对话功能

```
用户故事: 作为用户，我可以发送消息给鸠摩罗什Bot，获得佛学相关的智能回复

技术实现:
- 接入DeepSeek API (streaming模式)
- 构建Prompt模板，注入鸠摩罗什人格
- 处理用户输入，过滤敏感内容

验收标准:
✓ 能在5秒内返回回复
✓ 回复内容符合鸠摩罗什人格设定
✓ 支持文字对话
✓ 基础错误处理
```

### 2. 佛经知识库

```
用户故事: 作为用户，我可以询问佛经相关问题，获得准确的解答

技术实现:
- 已有的6部佛经数据
- 构建向量检索/关键词检索
- 返回原文引用+解读

支持的6部佛经:
1. 金刚经 (鸠摩罗什译)
2. 心经 (鸠摩罗什译)
3. 维摩诘经
4. 妙法莲华经
5. 阿弥陀经
6. 楞严经 (部分)

验收标准:
✓ 能回答佛经基础问题
✓ 能引用原文
✓ 准确率>80%
```

### 3. 鸠摩罗什人格系统

```
用户故事: 与Bot对话时，感觉是在与鸠摩罗什大师对话

技术实现:
- soul.md人格设定文件
- 系统Prompt注入
- 对话风格引导

人格特点:
- 慈悲智慧
- 深入浅出
- 善于用比喻
- 温和包容

验收标准:
✓ 对话风格符合鸠摩罗什设定
✓ 能以大师角度回答问题
✓ 不会出现人格崩塌
```

### 4. 对话历史机制

```
用户故事: Bot能记住我们之前的对话，对话是有连续性的

技术实现:
- session内上下文保持
- 历史对话摘要存储
- 支持跨session记忆(可选)

技术方案:
- mem0 记忆系统
- 或自定义实现

验收标准:
✓ 同一会话内上下文连续
✓ 能回顾之前对话要点
✓ 超过20轮自动压缩
```

## P1 - 重要功能 (MVP后迭代)

### 5. 每日一偈

```
用户故事: 每天收到一句偈语及其解读，滋养心灵

功能描述:
- 每日定时推送(可选)
- 偈语库随机选取
- 简短解读(50-100字)
- 支持分享

实现方式:
- 偈语数据库
- 定时任务/推送服务

验收标准:
✓ 每日更新
✓ 解读有深度
✓ 可分享到微信
```

### 6. 禅修指导

```
用户故事: 作为用户，我可以获得禅修方法指导

功能描述:
- 基础禅修知识问答
- 呼吸法指导
- 常见禅修问题解答
- 简单引导冥想(文字/音频)

知识库内容:
- 止观禅修基础
- 数息观
- 念佛禅
- 常见问题FAQ

验收标准:
✓ 能回答常见禅修问题
✓ 提供可操作的指导
✓ 语气温和鼓励
```

### 7. 上下文压缩

```
用户故事: 对话很长时，Bot能自动压缩上下文，不丢失重要信息

技术实现:
- 检测token数量
- 提取关键信息
- 生成压缩摘要
- 继续对话

触发条件:
- 剩余token < 阈值
- 用户确认压缩

验收标准:
✓ 自动触发
✓ 压缩后不丢失核心信息
✓ 对话继续流畅
```

## P2 - 增强功能 (后续迭代)

### 8. 用户画像系统

```
功能: 记录用户兴趣、修行进度、提问历史

数据维度:
- 关注的佛经/话题
- 禅修经验水平
- 提问频率
- 学习进度

应用:
- 个性化推荐
- 进阶内容解锁
```

### 9. 分享功能

```
功能: 分享精彩对话/偈语到微信

形式:
- 生成分享图片
- 链接分享
- 海报分享
```

### 10. 多轮引导对话

```
功能: 提供主题式对话引导

场景:
- 入门系列(10课)
- 某一经书解读系列
- 禅修进阶系列
```
